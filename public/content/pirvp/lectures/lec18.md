# Лекция №18. WebSocket

## 1. Введение в WebSocket

### 1.1. Что такое WebSocket

WebSocket — это протокол, который позволяет устанавливать постоянное соединение между клиентом и сервером. Главная особенность WebSocket — двусторонняя связь (full-duplex), то есть сервер может отправлять данные клиенту в любой момент, а клиент — серверу.

**Преимущества WebSocket:**  
- Мгновенная передача данных без необходимости многократных запросов.  
- Экономия ресурсов, так как нет постоянного открытия и закрытия соединений, как в HTTP.  
- Идеален для чатов, онлайн-игр, уведомлений и других real-time приложений.

**Пример ситуации:** в обычном HTTP клиент каждый раз отправляет запрос на сервер, чтобы узнать новые сообщения. С WebSocket сервер сам сообщает клиенту о новых сообщениях сразу после их появления.

### 1.2. Отличие от HTTP

HTTP — это протокол «запрос-ответ». Клиент отправляет запрос, сервер отвечает, и соединение закрывается.  
WebSocket же поддерживает постоянное соединение. Это значит:  
- Сервер может отправлять данные клиенту без запроса.  
- Клиент и сервер могут обмениваться сообщениями одновременно.

**Пример:**  
- **HTTP:** клиент спрашивает: «Есть новые сообщения?» → сервер отвечает → соединение закрыто.  
- **WebSocket:** сервер сам сообщает клиенту «Появилось новое сообщение!» в момент его появления.

### 1.3. Сценарии использования real-time

WebSocket идеально подходит для приложений, где важна мгновенная реакция:  
- Чаты и мессенджеры.  
- Онлайн-игры.  
- Системы уведомлений.  
- Совместная работа в реальном времени (редактирование документов, доски задач).

## 2. Socket.io: клиент и сервер

### 2.1. Что такое Socket.io

Socket.io — библиотека для Node.js и браузера, которая упрощает реализацию real-time коммуникации через WebSocket. Если WebSocket недоступен, Socket.io автоматически переключается на fallback-методы (например, long-polling).

**Плюсы использования Socket.io:**  
- Простое подключение клиента к серверу.  
- Поддержка событий в реальном времени (real-time).  
- Работа с комнатами и пространствами (rooms).  
- Надёжная работа даже при нестабильной сети.

**Пример сценария:** чат, уведомления, совместное редактирование текста.

### 2.2. Серверная часть (Express + Socket.io)

На сервере создаём Express-приложение и подключаем Socket.io.

```javascript
// server.js
import express from 'express';
import { createServer } from 'http';
import { Server } from 'socket.io';

const app = express();
const httpServer = createServer(app);
const io = new Server(httpServer, {
  cors: { origin: '*' } // разрешаем подключение с любого фронтенда
});

const PORT = 3001;

// В память храним сообщения
const messages = [];

io.on('connection', (socket) => {
  console.log(`Клиент подключился: ${socket.id}`);

  // Отправляем историю сообщений новому клиенту
  socket.emit('chatHistory', messages);

  // При получении нового сообщения
  socket.on('sendMessage', (msg) => {
    messages.push(msg);          // сохраняем в массив
    io.emit('receiveMessage', msg); // рассылаем всем клиентам
  });

  socket.on('disconnect', () => {
    console.log(`Клиент отключился: ${socket.id}`);
  });
});

httpServer.listen(PORT, () => {
  console.log(`Сервер запущен на http://localhost:${PORT}`);
});
```

**Объяснение:**  
- `io.on('connection', callback)` — вызывается, когда клиент подключается.  
- `socket.emit()` — отправка данных конкретному клиенту.  
- `io.emit()` — отправка данных всем подключённым клиентам.  
- `messages` хранит все сообщения в памяти для примера.

### 2.3. Клиентская часть (React + Socket.io-client)

На фронтенде подключаемся к серверу и обрабатываем события.

```jsx
import React, { useEffect, useState } from 'react';
import { io } from 'socket.io-client';

const socket = io('http://localhost:3001');

export default function Chat() {
  const [messages, setMessages] = useState([]);
  const [input, setInput] = useState('');

  useEffect(() => {
    socket.on('chatHistory', (history) => {
      setMessages(history);
    });

    socket.on('receiveMessage', (msg) => {
      setMessages((prev) => [...prev, msg]);
    });

    return () => {
      socket.off('chatHistory');
      socket.off('receiveMessage');
    };
  }, []);

  const sendMessage = () => {
    if (!input) return;
    socket.emit('sendMessage', input);
    setInput('');
  };

  return (
    <div style={{ maxWidth: '500px', margin: '0 auto', padding: '20px' }}>
      <h2>Чат в реальном времени</h2>
      <div style={{ border: '1px solid #ccc', padding: '10px', height: '200px', overflowY: 'auto', marginBottom: '10px' }}>
        {messages.map((msg, index) => (
          <div key={index}>{msg}</div>
        ))}
      </div>
      <input
        value={input}
        onChange={(e) => setInput(e.target.value)}
        placeholder="Введите сообщение"
        style={{ width: '70%', marginRight: '5px' }}
      />
      <button onClick={sendMessage} style={{ width: '25%' }}>Отправить</button>
    </div>
  );
}
```

**Объяснение:**  
- `useEffect` подписывается на события `chatHistory` и `receiveMessage`.  
- `socket.emit('sendMessage', input)` отправляет сообщение на сервер.  
- `setMessages` обновляет состояние и перерисовывает чат.  
- Всё работает без базы данных, сообщения хранятся в памяти.

### 2.4. Логика работы чата на Socket.io

Когда пользователь открывает страницу чата, происходит следующее:

1. **Подключение клиента**  

- React-компонент создаёт соединение с сервером через `io('http://localhost:3001')`.  
- Сервер регистрирует подключение через `io.on('connection', callback)` и выдаёт каждому клиенту уникальный `socket.id`.

2. **История сообщений**  

- Сервер сразу отправляет новому клиенту массив `messages` через событие `chatHistory`.  
- Клиент получает массив и отображает все предыдущие сообщения.

3. **Отправка нового сообщения**  

- Пользователь вводит текст и нажимает «Отправить».  
- Клиент вызывает `socket.emit('sendMessage', input)`, отправляя сообщение на сервер.  
- Сервер добавляет сообщение в массив и рассылает всем клиентам через `io.emit('receiveMessage', msg)`.  
- Каждый клиент получает событие `receiveMessage` и обновляет отображение чата.

4. **Отключение клиента**  

- При закрытии вкладки или потере соединения срабатывает событие `disconnect`.  
- Сервер логирует отключение, но сообщения остаются в памяти для других пользователей.

**Ключевые моменты для понимания работы событий:**  
- `io.on('connection')` — фиксирует подключение каждого клиента.  
- `socket.emit()` — отправка данных конкретному клиенту.  
- `io.emit()` — рассылка события всем подключённым клиентам.  
- `socket.on()` — клиент или сервер слушает события и реагирует на них.
